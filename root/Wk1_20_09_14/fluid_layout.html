Fluid Layout

Transferring your base css to fluid is simple enough.

The container element is changed to 90% of the viewport width. This allows either a stretch out (for very large screens) or a collapse in (for smaller screens)

To see the difference from the fixed layout, minimise your screen.

The elements now alter according to the width of the viewport.

Problems with this are that with this solution the layout is no longer strict and so for different viewports the view is different.

Heading text elements do not respond well to this system

Rely heavily on media rules

Have to use alot of !important statements to override css

@media rules

@media are a set of rules that you can apply to your css file that select a certain element and alter its attributes to your needs

e.g. the rule @media all {the element you want to target}{css to go here}

Using these rules we can change the behavior of the elements on the page.

Excersise

Open the file at_media.txt

copy this to your fluid_styles.css file, save and close.

Refresh your browser and minimise to see the changes.
